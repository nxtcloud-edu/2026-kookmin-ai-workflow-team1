# 🍚 밥BTI — 팀 작업노트 (Handoff)

> **이 문서 용도:** 팀원(과 팀원의 AI)이 지금까지 진행 상황을 빠르게 파악하고 **바로 이어서 작업**할 수 있게 정리한 인수인계 문서입니다. 카톡으로 전달받았다면, 같은 폴더의 `index.html`이 현재 결과물입니다.

---

## 1. 프로젝트 한 줄 요약
대학 단톡방 인원들의 **음식 취향(좋아하는 음식·매운맛·예산·식사량)**을 입력받아,
취향이 잘 맞는 **밥메이트 3~4명**으로 묶고 **최적의 식당**을 추천하는 웹 서비스 = **밥BTI**.

- **AWS 부트캠프 하루짜리 팀 프로젝트** (국민대 AI Workflow, 팀1).
- **제출물 = HTML**, 사람들이 각자 기기로 **링크 들어와 체험 + 인기투표**. **1등 상품 = 마우스**.
- 그래서 핵심 승부처는 기술 깊이보다 **"열어본 사람이 재밌어서 표 주고 싶은가"**. → 즉석 체험 + 결과 카드 + 재미 요소 중심.

## 2. 확정된 방향 / 결정 (왜 그렇게 했는지)
| 결정 | 이유 |
|---|---|
| **동아리 빼고 학과만** | 하루 스코프, 복잡도↓ |
| 추천 알고리즘: **ALS/SVD++ ❌ → 코사인/항목별 유사도 가중합 + 그리디 그룹핑 ✅** | 취향을 **한 번만** 입력 = 상호작용 이력 0 = **콜드스타트**라 행렬분해가 안 맞음. 발표 때 "왜 코사인 골랐나"를 설명하면 오히려 더 똑똑해 보임 |
| 아키텍처: **단일 `index.html`(클라이언트 사이드) + Firebase Realtime DB 실시간 레이어** | 서버 없이 어디든 올리면 동작 → 투표 링크로 안 죽음. 실시간 전체 매칭만 Firebase로 |
| 호스팅: **S3 정적호스팅(부트캠프 어필) 또는 GitHub Pages** | 투표 링크는 항상 떠 있어야 함. ngrok은 터널 끊겨서 부적합 |

## 3. 현재 결과물 (`index.html`) — 무엇이 되는가
**단일 HTML 파일 하나**로 아래가 전부 동작 (브라우저로 더블클릭하면 바로 확인 가능):
1. **입력폼**: 룸 코드 / 닉네임 / 좋아하는 음식(멀티) / 매운맛(0~5) / 예산(6천~1.6만) / 식사량(소·보통·대)
2. **밥BTI 타입**: 입력 기반으로 음식 취향 타입 부여 (예: 🔥매운맛 헌터, 💸가성비 미식가, 🕯️분위기파, 🥩육식파, 🌏잡식 탐험가 …)
3. **밥메이트 추천**: 가상 학과 단톡방 **샘플 15명**과 유사도 매칭 → 상위 3명 + **일치율 %** + **매칭 이유**("한식·분식 취향 일치, 매운맛 코드 맞음")
4. **그룹 추천 식당**: 나+밥메이트 그룹의 평균 취향으로 학교 앞 식당 큐레이션에서 3순위 추천
   - **📍 메뉴 → 주변 실제 식당 (토글 + GPS)**: 추천 메뉴 카드의 "📍 주변 실제 식당 보기"를 누르면(토글), 현재 위치 주변에서 **그 메뉴에 맞는 실제 식당**이 펼쳐짐. OpenStreetMap Overpass API(키 불필요)로 반경 1km 검색 → **식당 이름 키워드 매칭**(한국 식당명에 메뉴가 들어감: 박씨돈까스·신전떡볶이 등) 1순위 + 카테고리/cuisine 보조 + 거리순. 위치 1회만 받아 캐시. 권한 거부·실패 시 안내(메뉴 카드는 유지).
5. **🔴 라이브 룸 (실시간 전체 매칭)**: 같은 룸 코드로 들어온 **실제 참여자**들을 실시간으로 3~4명씩 자동 그룹핑 → *Firebase 설정 시 활성화* (미설정이면 "단일 기기 데모 모드"로 안내, 안 깨짐)
6. **반 취향 통계**: 매운맛 분포 / 인기 음식 TOP 막대그래프

> ✅ Playwright로 입력→결과 전체 플로우 동작 검증 완료. 통계 막대가 안 채워지던 버그(`.bar-fill`이 inline이라 width 무시) → `display:block`으로 수정 완료.

## 4. 코드 구조 (전부 `index.html` 한 파일 안)
- `<style>`: 디자인(따뜻한 오렌지 톤, 카드 UI, 모바일 반응형)
- `CATS`: 음식 카테고리 9종 / `PEOPLE`: 데모용 샘플 15명 / `MENUS`: 추천 메뉴(음식) 12종 — 가상 식당이 아니라 "삼겹살 구이·해물 짬뽕·돼지 두루치기" 같은 메뉴 이름
- `decideType(u)`: 밥BTI 타입 판정 (우선순위 규칙 기반)
- `similarity(a,b)`: 두 사람 유사도 = 카테고리 자카드 0.4 + 매운맛 0.3 + 예산 0.18 + 식사량 0.12 가중합, + 매칭 이유 문구 생성
- `recommendRestos(group)`: 그룹 평균 취향 ↔ 메뉴(음식) 속성 매칭(콘텐츠 기반) → 추천 메뉴 TOP3. 데이터는 `MENUS`(삼겹살 구이·해물 짬뽕 등 음식 이름, 가상 식당 아님)
- `fetchNearbyRestos(lat,lon,radius)` + `osmToResto` + `haversine` + `loadNearby`(위치 1회 획득·캐시) + `matchRestosForMenu`(메뉴별 이름 키워드 매칭) + `loadMenuRestos`(토글 펼침 시 렌더): **📍 메뉴→주변 실제 식당**. Overpass API로 주변 음식점 검색 → 식당 이름에 메뉴 키워드(`MENUS[].kw`)/카테고리 키워드(`CAT_KEYWORDS`)/cuisine 태그 매칭 점수(+100/+60/+40) + 거리순. `renderRestos`가 각 메뉴를 토글 카드(`.resto-head`/`.resto-detail`)로 렌더. GPS 실패 시 원인별 메시지 + `DEFAULT_LOC`(국민대) 폴백.
- `formGroups(members, size)`: 그리디 그룹핑(유사도 높은 사람끼리 묶기) — 라이브 룸용
- Firebase: 상단 `firebaseConfig`(현재 `PASTE_` 자리표시자) — 설정하면 실시간 켜짐

## 5. ⏳ 남은 작업 (우선순위)
1. **Firebase 실시간 켜기** ← 한 사람이 5분이면 함
   - [console.firebase.google.com](https://console.firebase.google.com) → 프로젝트 만들기
   - 빌드 > **Realtime Database** > 데이터베이스 만들기 > **테스트 모드로 시작**
   - ⚙️ 프로젝트 설정 > 내 앱 > 웹(`</>`) 등록 > `firebaseConfig` 복사
   - `index.html` 상단 `firebaseConfig`의 `PASTE_...` 4줄 교체 (**`databaseURL` 꼭 포함**)
   - 테스트: 2개 탭/기기에서 같은 룸코드(`lunch`)로 제출 → "지금 2명 참여 중" + 그룹 뜨면 성공
2. **배포**: `index.html`을 **S3 정적 웹호스팅** 또는 **GitHub Pages**에 올려 투표용 링크 만들기
3. **표심 강화(여유 시)**: 결과를 이미지 카드로 저장/공유(스샷각), ~~식당 데이터를 실제 학교 앞 맛집으로 교체~~ → **✅ GPS 주변 실제 식당 검색으로 대체 구현됨** (Overpass API), 통계 비주얼 강화
4. **발표 준비**: "왜 코사인/클러스터링을 골랐나(콜드스타트)" 1장, 데모 시나리오(전원이 같은 룸코드로 동시 입력 → 실시간 그룹 형성)

## 6. 분담 제안 (3명)
1. **데이터 수집/입력 UI** — 입력폼, 데이터 포맷, (원하면) 구글폼 연동
2. **추천/매칭 알고리즘** — `similarity`/`recommendRestos`/`formGroups` 튜닝, 식당·타입 데이터 보강
3. **시각화/결과 카드/배포** — 결과 카드(스샷각), 통계 비주얼, S3·Pages 배포 ← **표를 제일 많이 부르는 파트**

## 7. 실행 방법
- **빠른 확인**: `index.html` 더블클릭 (디자인·개인 체험·통계까지 다 보임. 단, 실시간은 안 됨)
- **실시간 테스트**: VS Code Live Server로 열거나 배포 후 (file:// 로는 Firebase가 막힐 수 있음)

## 8. 주의
- Firebase 웹 `apiKey`는 **공개돼도 되는 값**(보안은 DB 규칙으로). 단 테스트 모드는 누구나 쓰기 가능 → **데모 끝나면 프로젝트 삭제하거나 규칙 잠그기**.
- 라이브 룸은 공개 링크라 닉네임 **XSS 이스케이프** 이미 처리됨.
